## 📖 О проекте

**QA Sprint Board** — это лёгкая фронт-доска спринта, заточенная под повседневную работу линейного тестировщика. Никаких сложных дашбордов: открыл и за пару секунд понял картину дня.

> **Бэкенда нет — это чисто фронт.** Всё состояние живёт в одном файле
> [`src/data/sprint.ts`](src/data/sprint.ts). Доску обновляет лид (через ассистента)
> каждое утро под актуальное распределение и проценты из отчётов QA.

## ✨ Возможности

- **Две команды в одном месте** — переключатель **CORE / eQA** (у внешней команды видны часы смен).
- **Режимы просмотра** — «Вся команда», «Можно взять» и фокус на конкретном тестере. На всех кнопках — счётчики карточек.
- **Критбизнес наверху** — задачи, которые брать первым, закреплены отдельной полосой.
- **Двойная шкала прогресса** на каждой карточке — «Тесты по чек-листу на stage» и «Ретесты на stage», чтобы было видно, на каком этапе эпик.
- **Готовность спринта** — автоматический расчёт по всем эпикам команды + темп («успеваем / впритык / отстаём») относительно прошедшего времени.
- **Граф связей эпика** — клик по карточке открывает живой force-directed граф всех дочерних и связанных задач с цветовой картой статусов, счётчиками и фильтром по цвету.
- **Статусы 1:1 с Jira** — бейджи повторяют реальный воркфлоу (`Analysis`, `Блок тесты`, `R.F. QA`, `QA testing`, `R.F Release`, `Готово` …).
- **Адаптив и safe-area** — корректно живёт на мобайле (от 360px) и устройствах с вырезом.

Отдельная вкладка для внешней команды — кто над чем работает, с часами смен.

![Команда eQA](docs/screenshots/eqa.png)

### Граф связей эпика
Клик по карточке → живой граф. В центре эпик, вокруг — дочерние (`parent`) и связанные (issue links) задачи. Цвет узла = смысл статуса, баги крупнее, связанные — с белой обводкой.

### Фильтр графа по цвету
В легенде у каждого статус-цвета счётчик задач. Клик по цвету оставляет только нужные узлы — можно комбинировать (например, только reopen + новые).

## 🗂️ Архитектура

```text
src/
├─ data/
│  ├─ sprint.ts        # ЕДИНЫЙ источник: эпики, команда, смены, распределение
│  ├─ epicGraph.ts     # типы + цветовая карта тонов графа
│  └─ epicGraph.json   # снапшот связей эпиков (генерируется из Jira)
├─ lib/
│  └─ format.ts        # метаданные статусов/приоритетов, прогресс спринта, аватары
├─ components/
│  ├─ Avatar.tsx
│  ├─ Badges.tsx       # StatusBadge, PriorityBadge
│  ├─ EpicCard.tsx     # карточка эпика (двойная шкала, ссылки, граф по клику)
│  └─ EpicGraphModal.tsx  # модалка force-directed графа + фильтр по цвету
└─ app/
   ├─ page.tsx         # витрина: зоны, режимы, шкала готовности (client component)
   ├─ layout.tsx       # metadata, viewport (safe-area)
   └─ globals.css

scripts/
└─ genGraph.mjs        # генератор epicGraph.json из сырых дампов Jira
```

### Модель данных (`src/data/sprint.ts`)

| Сущность | Что описывает |
|---|---|
| `epics` | эпики: ключ, название, цель, приоритет, `jiraStatus`, `progress {firstPass, retest}`, ссылки и флаги (`critbusiness`, `bonus`, `goalDone`, `task`) |
| `assignments` | кто над чем работает: `{ memberId, epicKeys[], note }` |
| `members` | состав команд, отпуска (`onVacation`), смены eQA (`shift`) |
| `sprint` | номер, даты, ссылка на Confluence |

**Логика зон:** эпик команды, который не встречается ни в одном `assignment`, автоматически попадает в **«Можно взять»**. Критбизнес закрепляется сверху главного экрана.

## 🔄 Как обновлять данные

Утром лид присылает отчёты/правки → доска приводится в актуальное состояние:

1. **Статусы эпиков** — сверяются с live-Jira (`searchJiraIssuesUsingJql`, `key in (…)`), маппинг имени статуса → enum в [`lib/format.ts`](src/lib/format.ts).
2. **Проценты** — из ежедневных отчётов QA (Slack — первоисточник). Источник правды — формулировка самого тестера; дробь не пересчитывается.
3. **Граф эпиков** — снапшот `epicGraph.json` перегенерируется из дампов Jira:

```bash
# дочерние задачи общим дампом + связанные одним запросом
node scripts/genGraph.mjs --reset \
  --children-grouped kids1.json kids2.json … \
  --links links.json
```

> Подробный контекст для ассистента — в [`AGENTS.md`](AGENTS.md), журнал правок — в [`CHANGELOG.md`](CHANGELOG.md).
