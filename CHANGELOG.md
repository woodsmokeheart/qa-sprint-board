## 18.06.2026 — производительность графа

- Граф подтормаживал при активной вкладке (у автотестов «гладко», т.к. вкладка
  в фоне и rAF дросселируется). Причина — `ctx.shadowBlur` на каждой из 155–165
  нод × каждый кадр (частицы на рёбрах держат перерисовку 60fps). shadowBlur —
  одна из самых тяжёлых canvas-операций.
- Фикс: убрал `shadowBlur`, заменил на дешёвое «гало» (полупрозрачный круг под
  нодой) — выглядит так же, но без блюра. Добавил `warmupTicks` (граф
  открывается уже устаканенным) и поднял `d3VelocityDecay` (быстрее затухает).
- Визуал сохранён, FPS на тяжёлых эпиках больше не проседает.

## 18.06.2026 — граф эпика (force-directed)

**Новое: клик по карточке эпика открывает живой граф связей.**
- Движок: **`react-force-graph-2d`** (canvas, физика d3-force) — грузится через
  `next/dynamic` с `ssr:false`. Свечение нод (canvas shadowBlur), частицы на
  рёбрах, зум колесом, драг узла, клик по ноде → задача в Jira, тултип по hover.
- В центре эпик, вокруг — **все дочерние задачи** (Задачи + Баги) нодами.
  Цвет узла = смысл статуса (`toneOfStatus`): reopen/блок — розовый, новые —
  янтарь, в QA — голубой, release/merge — бирюза, готово — зелёный. Баги крупнее.
- Данные — снапшот из Jira в `src/data/epicGraph.json`, собирается скриптом
  `scripts/genGraph.mjs` из сырого дампа (`parent = KEY`, MCP). Бэка нет —
  перегенерируется ассистентом вместе с утренними правками.
- Снапшот собран по **всем 25 эпикам** доски. Эпики без детей (BF-2428,
  BF-3231, BF-3160, BF-3193, BR-18 — одиночные) показывают плашку «снапшот
  ещё не собран».
- **Пагинация:** Jira отдаёт максимум 100 задач за страницу. genGraph.mjs
  теперь мёржит несколько дампов на эпик и дедуплицирует по ключу, так что
  крупные эпики берутся целиком. Пример: BF-2209 = 155 дочерних (100+55),
  раньше граф врал «100». Остальные эпики были < 100 — у них всё полно.
- **Связанные задачи (issue links) добавлены в граф.** Кроме дочек (`parent`)
  граф теперь показывает связанные через issue links — это отдельный тип связи,
  туда же попадают кросс-проектные «дети» (BF-эпик → SD-таски, тип Parent-Child).
  Связанные ноды — с фиолетовой обводкой и пунктирным ребром; в шапке отдельный
  счётчик. Пример: BF-2209 = 155 дочек + 10 связанных = 165.
- Сбор связей: один запрос `key in (все эпики)` с `fields=["issuelinks"]`,
  genGraph.mjs в режиме `--links` группирует по ключу issue и дедуплицирует
  (исключая то, что уже есть среди дочек). Эпики без детей, но со связями
  (BF-3193, BR-18, BF-2428…) теперь тоже имеют граф.
- Модель: `EpicGraphData.linked?: LinkedNode[]` (с полем `relation`).
- **Аудит полноты.** Прогнал ground-truth сверку: `parent in (все эпики)`
  постранично (10 страниц, 961 ребёнок), сгруппировал по родителю и сравнил
  со снапшотом. Результат — **0 расхождений** по всем 29 эпикам. Связи берутся
  из непагинируемого `issuelinks`, поэтому тоже полные. Снапшот актуален.
- Карточка кликабельна целиком (открывает граф); ссылка на Jira осталась на
  номере тикета (`stopPropagation`). Закрытие: X / клик по фону / Esc.

**Файлы:** `src/data/epicGraph.ts` + `src/data/epicGraph.json` (данные),
`scripts/genGraph.mjs` (генератор), `src/components/EpicGraphModal.tsx`,
правки `src/components/EpicCard.tsx`. Зависимость: `react-force-graph-2d`.

> Известный момент (не из этой задачи): в dev горит overlay hydration-warning
> из `Preloader` (`page.tsx:338`, `Date.now()/Math.random()`) — на прод-сборку
